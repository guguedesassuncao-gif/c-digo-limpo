# codigo-limpo
Resumo — Código Limpo (Capítulos 2, 4 e 5)

O livro Código Limpo, de Robert C. Martin, ensina boas práticas para criar códigos mais organizados e fáceis de entender.

Capítulo 2 — Nomes Significativos

O autor explica que variáveis, funções e classes devem ter nomes claros e objetivos. Bons nomes facilitam a leitura do código e evitam confusão.

Importância:
Melhora a compreensão do sistema e facilita manutenção.



Capítulo 4 — Comentários

Comentários devem ser usados apenas quando realmente necessários. Um código bem escrito já deve ser fácil de entender sozinho.

Importância:
Evita confusão e torna o código mais limpo.



Capítulo 5 — Formatação

A organização visual do código é importante. Indentação correta e separação adequada ajudam na leitura.

Importância:
Facilita o trabalho em equipe e melhora a legibilidade.



Conclusão

Os capítulos mostram que escrever código limpo ajuda a reduzir erros, melhorar a manutenção e facilitar o entendimento do sistema por outros desenvolvedores.
